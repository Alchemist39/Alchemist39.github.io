1) делаем игру-кликер для развлеения и самообучения.
2) игра на локальной машине, без логинов и паролей, однопользовательская.
3) Общее: основной экран делится на две одинаковые части. Слева находится враг или враги и ничего 
более. Справа содержатся элементы управления и отображения.

Враг: сущность, состоящая из 
-- тела (картинка), 
-- полоски здоровья, которая в свою очередь состоит из
	-- рамки, 
	-- полоски здоровья, уменьшающейся в зависимости от оставшегося количества здоровья врага, 
 	-- цифры, отображаюющей текущий уровень здоровья.

 Уровень: 
 -- сущность, отображающая текущий прогресс игрока.

 Золото: 
 -- сущьность, отображающая и хранящая текущее количество золота игрока.

 Ресет: 
 -- кнопка обнуления текущего прогресса игрока (с картинкой), 
 	-- при нажатии на которую выводится сообщение о подтверждении действия и при согласии игрока 
 	-- очищается локальное хранилище.

 Герой: сущность состоящая из: 
 --картинки (для каждого героя своя),
 -- цифры, отображающей текущее количество героев данного типа,
 -- текущей стоимости покупки следущего героя.

 Действия: 
 1) при клике на врага его хп уменьшается на 5, при этом оно не может опуститься ниже нуля ни 
 при каких условиях. Когда текущее хп врага опускается до нуля происходит следущее:
-- добавляется золото в кошелек
-- исчезает враг
-- уровень увеличивается на один
-- появляется новый враг с новыми случайными координатами, с полным хп, количество которого
 	равно 3 * текущий уровень. 

 2) при клике на героя происходит:
 -- проверка текущего количества золота, если его меньше текущей цены героя, в консоль выводится
 	сообщение о нехватке золота.
 -- если золота достаточно, то:
 	-- из кошелька удаляется золото равное текущей стоимости героя
 	-- количество героев увеличивается на один
 	-- стоимость следущего героя увеличивается до 1.25 от текущего количества героев данного типа

 3) отображение: 
 -- враг, уровень, золото, ресет, первый герой отображаются всегда. 
 -- Остальные герои по  умолчанию скрыты. 
 -- В момент, когда количество золота больше или равно начальной стоимости следущего  героя, он 
 	становится видимым.  
 -- Если денег становится меньше, чем стоимость героя, он становится полупрозрачным
 -- Если денег вновь хватает для покупки героя, он вновь становится непрозрачным.

 4) Сохранение:
 -- в локальное хранилище сохраняются: уровень, золото, количество купленных героев.
 -- время выхода из игры для выссчитывания золота, полученного за время оффлайна.

 		-- если денег не хватает, картинка полупрозрачная
 		-- если денег хватает, картинка непрозрачная

 	Общие улучшения:
 	-- увеличивают урон каждого героя на N %
 	-- увеличивают награду за убийство противника на N %
 	-- улучшения друг с другом перемножаются

 	Личные улучшения:
 	-- увеличивают урон только текущего героя, не влияя на остальных
